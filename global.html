skate/transact

transact() is Skate's core implementation of IndexedDB transactional injection, which is the process of introspecting a function's parameters, creating a transaction to suit them, and then calling the function, passing objects to the function as it requested via naming.

This is function injection in a way popularized by the Angular.js framework. transact() is used internally within Skate in many places, including Database.transact(), SchemaBuilder.run() and Repository.hydrate().

transact(transactionOrFactory, repositoryFactory, fn, mode, extraInjectables) → {Promise}

Calls a function, injecting the proper repositories and related services (db, transaction), for the given transaction (or the transaction created by a factory function). If no transaction is passed, one is automatically created based on the repositories requested by the given function.

Parameters:
IDBTransaction | function

The transaction to use, or a function to construct a transaction from an array of object store names and a mode (ie function(storeNames, mode))

function

function

The function which should be introspected and run

String

The transaction mode ('readonly' or 'readwrite')

String

An object containing extra services which should be made available for injection in addition to the standard ones
